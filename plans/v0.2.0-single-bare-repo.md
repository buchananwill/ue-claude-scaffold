# v0.2.0 — Single Persistent Bare Repo

## Context

The current git architecture has two modes: a legacy single bare repo (`bareRepoPath`) and a V3
per-agent bare repo layout (`bareRepoRoot/{agentName}.git`). Both are overcomplicated. Git branches
are lightweight refs, not separate databases — multiple agents can push to different branches in the
same bare repo without object-level conflicts. The per-agent bare repo design and the fragile
sync dance in `launch.sh` exist because this wasn't understood when the scaffold was first built.

### Goal

Collapse to exactly **two repos**:

1. **User repo** (non-bare) — the interactive session's git, with worktrees like `PistePerfect_5_7`,
   `claude_work`, `coralie_work`. Off-limits to containers (defense-in-depth).
2. **Pipeline bare repo** (bare) — single persistent `.git` database shared by all container agents.
   Created once, incrementally synced thereafter.

### Branch model

```
main                          ← project-canonical stable branch (lives in user repo)
docker/current-root           ← integration branch: what containers fork from on fresh start
docker/agent-1                ← agent 1's working branch
docker/agent-2                ← agent 2's working branch
```

- The interactive session controls `docker/current-root` — merging in user changes and
  reviewed container work.
- Containers fork from `docker/current-root` and push to `docker/{agent-name}`.
- New plans committed to `docker/current-root` can be merged into agent branches without
  resetting containers.

### Per-agent staging worktrees are retained

The build/test endpoint continues to use per-agent staging worktrees (one checked-out directory per
agent for host-side builds). These are checked out from the single bare repo. Simplifying staging
worktrees is deferred to v0.3.0.

```typescript
  // ... rest unchanged

```

## Phase 3 — Task route: push plans to agent branches

**Files:** `server/src/routes/tasks.ts`

### Current behaviour

`writeContentToBareRepo()` commits the plan to `config.tasks.planBranch` (default:
`docker/current-root`). This is correct — plans belong on the canonical branch.

### New behaviour: merge into agent branches

After writing the plan to `docker/current-root`, merge that commit into the target agent branches
so running containers can see it without a manual sync.

Add a `targetAgents` field to `TaskBody`:
```typescript
interface TaskBody {
  title: string;
  description?: string;
  sourcePath?: string;
  sourceContent?: string;
  acceptanceCriteria?: string;
  priority?: number;
  files?: string[];
  targetAgents?: string[];   // NEW
}
```

After the plan is committed to `docker/current-root`, for each target agent branch that exists:

```typescript
function mergeIntoBranch(bareRepoPath: string, sourceBranch: string, targetBranch: string): void {
  // Check target branch exists
  try {
    execFileSync('git', ['-C', bareRepoPath, 'rev-parse', '--verify', `refs/heads/${targetBranch}`], {
      stdio: 'ignore', timeout: 5000,
    });
  } catch {
    return; // Branch doesn't exist yet — agent hasn't started. Skip.
  }

  // Merge source into target using plumbing (no checkout needed)
  // Get the trees and create a merge commit
  const sourceRef = execFileSync('git', ['-C', bareRepoPath, 'rev-parse', `refs/heads/${sourceBranch}`], {
    encoding: 'utf-8', timeout: 5000,
  }).trim();
  const targetRef = execFileSync('git', ['-C', bareRepoPath, 'rev-parse', `refs/heads/${targetBranch}`], {
    encoding: 'utf-8', timeout: 5000,
  }).trim();

  // Check if already up-to-date (source is ancestor of target)
  const mergeBase = spawnSync('git', ['-C', bareRepoPath, 'merge-base', sourceRef, targetRef], {
    encoding: 'utf-8', timeout: 5000,
  });
  if (mergeBase.status === 0 && mergeBase.stdout.trim() === sourceRef) {
    return; // Already up-to-date
  }

  // Fast-forward if possible (target is ancestor of source)
  if (mergeBase.status === 0 && mergeBase.stdout.trim() === targetRef) {
    execFileSync('git', ['-C', bareRepoPath, 'update-ref', `refs/heads/${targetBranch}`, sourceRef], {
      timeout: 5000,
    });
    return;
  }

  // True merge needed — use read-tree to merge trees
  // This is the non-trivial case. Use git merge-tree (available since git 2.38)
  // to compute the merge without a worktree.
  const mergeResult = spawnSync('git', [
    '-C', bareRepoPath, 'merge-tree', '--write-tree', targetRef, sourceRef,
  ], { encoding: 'utf-8', timeout: 10000 });

  if (mergeResult.status !== 0) {
    // Merge conflict — don't force it. Log a warning but don't fail the task creation.
    // The operator will need to merge manually.
    throw new Error(`Merge conflict merging ${sourceBranch} into ${targetBranch}`);
  }

  const mergedTree = mergeResult.stdout.trim().split('\n')[0]; // First line is tree SHA
  const commitSha = execFileSync('git', [
    '-C', bareRepoPath, 'commit-tree', mergedTree,
    '-p', targetRef, '-p', sourceRef,
    '-m', `Merge ${sourceBranch} into ${targetBranch}`,
  ], { encoding: 'utf-8', timeout: 5000 }).trim();

  execFileSync('git', ['-C', bareRepoPath, 'update-ref', `refs/heads/${targetBranch}`, commitSha], {
    timeout: 5000,
  });
}
```

Behaviour:
- Always write plan to `docker/current-root` (canonical integration branch).
- If `targetAgents` is provided (e.g. `["agent-1"]`), merge `docker/current-root` into each
  named agent's `docker/{agentName}` branch.
- If `targetAgents` is `"*"`, merge into all active agent branches (query `agents` table for
  agents with status not in `done`, `error`).
- If `targetAgents` is omitted, **no merge** — the plan lands on `docker/current-root` only.
  The operator controls when to propagate via explicit `targetAgents` or the `/agents/{name}/sync`
  endpoint. This keeps the default conservative: you decide when plans reach agents.
- Return merge results in the response: `{ id, ok, commitSha, mergedAgents: ["agent-1"], failedMerges: [] }`
- Merge conflicts are **non-fatal** — the task is still created, the plan is on
  `docker/current-root`, but the agent will need a manual sync or reset to see it.

Also add a standalone merge endpoint for manual use:

```
POST /agents/{name}/sync
```

Merges `docker/current-root` into `docker/{agentName}`. Useful when pushing plans without
creating a task, or for syncing after merging other agents' work into `docker/current-root`.

### Acceptance criteria

- `POST /tasks` with `sourceContent` + `targetAgents: ["agent-1"]` commits to
  `docker/current-root` and merges into `docker/agent-1`.
- `POST /tasks` with `sourceContent` + `targetAgents: "*"` merges into all active agent branches.
- `POST /tasks` with `sourceContent` and no `targetAgents` commits to `docker/current-root` only
  (no agent branch merge).
- Merge conflicts are reported but don't fail task creation.
- `POST /agents/{name}/sync` merges `docker/current-root` into the named agent's branch.
- `npm run typecheck` passes.
- Tests pass.

## Phase 4 — launch.sh: single bare repo, resume-by-default

**Files:** `launch.sh`

### Remove per-agent bare repo logic

Delete:
- `BARE_REPO_ROOT` variable and config read (line 105)
- `STAGING_WORKTREE_ROOT` requirement for `--parallel` (lines 198-203) — rethink parallel later
- Per-agent bare repo path derivation `${BARE_REPO_ROOT}/${AGENT_NAME}.git` (line 166)
- The entire parallel launch block (lines 327-362) that creates per-agent `.git` repos

The parallel launch block needs rethinking for the single-repo model, but that's mechanical —
all agents share the same `BARE_REPO_PATH`, each gets its own branch `docker/agent-N`.

### New branch logic

Replace the current `WORK_BRANCH` resolution (lines 153-163) and bare repo sync dance
(lines 272-317) with:

```bash
# Branch naming: docker/{agent-name}
AGENT_BRANCH="docker/${AGENT_NAME}"

# The branch containers fork from
ROOT_BRANCH="${ROOT_BRANCH:-docker/current-root}"

# Bare repo must exist (created by setup.sh, persistent)
if [[ ! -d "$BARE_REPO_PATH" ]]; then
  echo "Error: Bare repo not found at $BARE_REPO_PATH" >&2
  echo "Run ./setup.sh to create it, or create manually:" >&2
  echo "  git clone --bare <your-project> $BARE_REPO_PATH" >&2
  exit 1
fi

# Ensure docker/current-root exists in the bare repo
if ! git -C "$BARE_REPO_PATH" rev-parse --verify "refs/heads/${ROOT_BRANCH}" &>/dev/null; then
  echo "Error: Branch '${ROOT_BRANCH}' not found in bare repo." >&2
  echo "Push it from your project: git push $BARE_REPO_PATH HEAD:refs/heads/${ROOT_BRANCH}" >&2
  exit 1
fi

if [ "$_CLI_FRESH" = "true" ]; then
  # Fresh: create/reset agent branch from docker/current-root
  echo "Creating fresh branch ${AGENT_BRANCH} from ${ROOT_BRANCH}..."
  ROOT_SHA=$(git -C "$BARE_REPO_PATH" rev-parse "refs/heads/${ROOT_BRANCH}")
  git -C "$BARE_REPO_PATH" update-ref "refs/heads/${AGENT_BRANCH}" "$ROOT_SHA"
else
  # Resume (default): use existing agent branch, or create if first run
  if ! git -C "$BARE_REPO_PATH" rev-parse --verify "refs/heads/${AGENT_BRANCH}" &>/dev/null; then
    echo "No existing branch ${AGENT_BRANCH}. Creating from ${ROOT_BRANCH}..."
    ROOT_SHA=$(git -C "$BARE_REPO_PATH" rev-parse "refs/heads/${ROOT_BRANCH}")
    git -C "$BARE_REPO_PATH" update-ref "refs/heads/${AGENT_BRANCH}" "$ROOT_SHA"
  else
    echo "Resuming from existing branch ${AGENT_BRANCH}."
  fi
fi
```

The `WORK_BRANCH` env var passed to the container becomes `$AGENT_BRANCH`.

### Remove the sync dance

The entire block at lines 282-309 (adding `exchange` remote, fetch, ancestry check,
fast-forward-or-warn, force-push-back) is deleted. The bare repo is persistent and authoritative —
the user pushes to it explicitly. No bidirectional sync needed.

### Remove CLONE_SOURCE / staging worktree as clone source

Containers no longer clone from a staging worktree. They clone directly from the bare repo
(as they already do — `git clone /repo.git /workspace --branch $WORK_BRANCH`). The
`CLONE_SOURCE` variable and its resolution are removed from `launch.sh`.

### Update --parallel

The parallel block (lines 327-362) changes from creating per-agent bare repos to creating
per-agent branches in the single bare repo:

```bash
for i in $(seq 1 "$_CLI_PARALLEL"); do
  _AGENT="agent-${i}"
  _BRANCH="docker/${_AGENT}"

  # Create agent branch from docker/current-root if it doesn't exist
  if ! git -C "$BARE_REPO_PATH" rev-parse --verify "refs/heads/${_BRANCH}" &>/dev/null; then
    ROOT_SHA=$(git -C "$BARE_REPO_PATH" rev-parse "refs/heads/${ROOT_BRANCH}")
    git -C "$BARE_REPO_PATH" update-ref "refs/heads/${_BRANCH}" "$ROOT_SHA"
  fi

  # Launch container
  (cd "$SCRIPT_DIR/container" && \
    AGENT_NAME="$_AGENT" \
    WORK_BRANCH="$_BRANCH" \
    BARE_REPO_PATH="$BARE_REPO_PATH" \
    ... \
    $COMPOSE_CMD --project-name "claude-${_AGENT}" up --build --detach)
done
```

### Acceptance criteria

- `launch.sh` reads only `bareRepoPath` from config (no `bareRepoRoot`).
- Default mode is resume (uses existing `docker/{agent-name}` branch).
- `--fresh` resets `docker/{agent-name}` to `docker/current-root` HEAD.
- First run creates the agent branch from `docker/current-root`.
- No sync dance (no `exchange` remote, no fetch-from-bare, no force-push-back).
- `--parallel N` creates branches `docker/agent-1` through `docker/agent-N` in the single bare repo.
- `--dry-run` shows the new branch names.
- Bare repo must pre-exist (error if missing, with instructions).

## Migration for existing users

1. If you have per-agent bare repos (`bareRepoRoot`), the simplest migration is:
   - Pick one agent's bare repo (or the legacy `bareRepoPath`) as the single repo.
   - Create `docker/current-root` from its current HEAD.
   - Create `docker/agent-X` branches for each agent from their existing work.
   - Update `scaffold.config.json` to set `bareRepoPath` and remove `bareRepoRoot`.

2. Or `--fresh` to start clean:
   - Delete old bare repos.
   - Run `setup.sh` to create a fresh bare repo with `docker/current-root`.
   - All agents start from clean forks.

## Non-goals (deferred to v0.3.0)

- Simplifying per-agent staging worktrees (replacing with dynamic `git worktree add` from the
  single bare repo).
- Auto-coalesce tooling (merging agent branches back into `docker/current-root` via a server
  endpoint or script).
